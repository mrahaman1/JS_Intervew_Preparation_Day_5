2.What is a Temporal Dead Zone?

The Temporal Dead Zone is a behavior in JavaScript that occurs when declaring a variable with the let and const keywords, but not with var. In ECMAScript 6, accessing a let or const variable before its declaration (within its scope) causes a ReferenceError. The time span when that happens, between the creation of a variableâ€™s binding and its declaration, is called the temporal dead zone.

3.what is Symbol?

Symbol is a built-in object whose constructor returns a symbol primitive also called a Symbol value or just a Symbol that's guaranteed to be unique. Symbols are often used to add unique property keys to an object that won't collide with keys any other code might add to the object, and which are hidden from any mechanisms other code will typically use to access the object.

5.Explain passed by value and passed by reference

Pass By Value: In pass by value,function is called by directly passing the value of the variable as an argument.so any changes made inside the function does not effect the original value.In pass by value,parameters are passed as an arguments creates its own copy and changes made inside the function is made to copied value not to original value.

Pass By Reference:In Pass By reference function is called by directly passing the reference/address of the variable as an argument.so if any changes made inside the function also changes the original value.In JS,array and object follows pass by reference property.In,passby reference,parameters passed as an arguments does not create its own copy it refers to the original values so changes made inside function also affect original value.
